# Week 11 — Post-meeting AI

**Status:** ✅ green

## Goal

After a meeting ends, run a background job that produces:
1. a ≤200-word summary,
2. a list of decisions,
3. a list of action items with owner + deadline.

All via Ollama JSON-mode structured output. Idempotent — re-running on the same meeting overwrites prior outputs.

## What landed

- `backend/src/meetwit/llm/structured.py`
  - Pydantic schemas: `MeetingSummary`, `DecisionList`, `ActionItemList`, `ConflictList` (used in W12)
  - `structured_completion(...)` calls Ollama with `format: "json"`, parses + validates against the schema. On parse failure, returns a default-empty instance and logs (the pipeline keeps going).
- `backend/src/meetwit/services/post_meeting.py`
  - `process_meeting(meeting_id, engine, settings, model, progress)` walks: load transcripts → SUMMARY → DECISIONS → ACTIONS, writing to `summaries`/`decisions`/`action_items` tables. Updates `progress.stage` between phases for client polling.
- `backend/src/meetwit/routers/post_meeting.py`
  - `POST /post-meeting/{id}/process` → kicks off the job, returns `{process_id}`. Poll via `/knowledge/processes/{id}` (same registry as W8).
  - `GET /summaries/{id}` → meeting summary
  - `GET /decisions?meeting_id=&project=` → filter list
  - `GET /action-items?meeting_id=&owner=&status_filter=` → filter list
  - `PATCH /action-items/{id}` → status / owner / deadline edit
  - `GET /conflicts/{id}` → list conflicts (populated by W12)

## Manual smoke test

```bash
# Prereqs: meeting created, transcripts persisted, Ollama running
PID=$(curl -s -X POST http://127.0.0.1:5167/post-meeting/$MEETING_ID/process \
  -H 'content-type: application/json' \
  -d '{"model":"qwen2.5:7b-instruct"}' | jq -r .process_id)

# Poll
while true; do
  curl -s http://127.0.0.1:5167/knowledge/processes/$PID | jq .
  curl -s http://127.0.0.1:5167/knowledge/processes/$PID | jq -e .finished >/dev/null && break
  sleep 2
done

curl -s http://127.0.0.1:5167/summaries/$MEETING_ID | jq .
curl -s "http://127.0.0.1:5167/decisions?meeting_id=$MEETING_ID" | jq .
curl -s "http://127.0.0.1:5167/action-items?meeting_id=$MEETING_ID" | jq .
```
