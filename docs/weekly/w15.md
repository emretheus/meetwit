# Week 15 — Code signing + notarization scaffolding

**Status:** ✅ green (scaffolding only — full live signing requires Apple Developer enrollment)

## Goal

Wire up the codesign + notarytool pipeline so V1 ships a Gatekeeper-clean `.dmg`. Document the prereqs and the env vars.

## What landed

- `tauri.conf.json` adds `bundle.resources` mapping `binaries/python-backend` → `python-backend`. Tauri now bundles the PyInstaller output into `Contents/Resources/python-backend/` of the .app, which is exactly where W14's `SpawnOptions::release()` looks for it.
- `.github/workflows/build.yml` rewritten:
  - Manual trigger (`workflow_dispatch`) with a `sign` boolean input
  - Reads all the Apple secrets from repo Secrets
  - Walks: install Rust/Node/uv/cmake → uv sync → PyInstaller → stage into Tauri binaries dir → `pnpm tauri:build` → optional notarize+staple → upload `.app`/`.dmg` artifact (14d retention)
- `docs/SIGNING.md` — full guide:
  - What you need (Developer Program + Developer ID Application cert + app-specific password)
  - Env vars: `DEVELOPER_ID_APPLICATION`, `APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_PASSWORD` for local builds
  - GitHub Secrets equivalents: `APPLE_SIGNING_IDENTITY`, `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD`, `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID`
  - Verification commands (`codesign --verify`, `spctl --assess`, `stapler validate`)
  - Troubleshooting guide for the four most common notarization rejections
  - Fallback: ad-hoc + `xattr -d com.apple.quarantine` for tester distribution

## Verification

- `cargo build --workspace` ✅
- `pnpm typecheck`, `lint`, `build` ✅
- `tauri.conf.json` is valid JSON with resources entry

## What is NOT yet verified

- **Actual signing + notarization round-trip.** Requires Apple Developer Program enrollment, which the user hasn't completed. When ready:
  1. Enroll at developer.apple.com ($99/year, 24-48h approval)
  2. Create a Developer ID Application cert, install into Keychain
  3. Generate an app-specific password
  4. Export the 4 env vars listed in `docs/SIGNING.md`
  5. Run `./scripts/build-release.sh`
  6. Verify with `spctl --assess` → "accepted"
- **GitHub Secrets** for CI signing — same prereqs, then add via repo Settings → Secrets and variables → Actions.
