# Week 8 — Knowledge ingestion (RAG part 1)

**Status:** ✅ green

## Goal

Index local company documents end-to-end: walk a folder → parse PDF/DOCX/MD/TXT → chunk → embed with BGE-small-en-v1.5 → write to `documents` + `doc_chunks` + `doc_chunks_vec`. Background task with progress polling.

## What landed

- `backend/src/meetwit/indexing/`
  - `parser.py` — PyMuPDF (PDF), python-docx (DOCX with heading-aware sectioning), markdown-it-py (MD with heading tokens), native (TXT). Each parser returns `ParsedSection`s preserving page numbers and section titles.
  - `chunker.py` — sentence-aware recursive splitter, 500-token target + 100-token overlap, deterministic output.
  - `embedder.py` — `Embedder` class wrapping sentence-transformers, lazy model load, thread-safe, returns `NDArray[float32]` shape `(N, 384)`.
- `backend/src/meetwit/services/`
  - `indexer.py` — `index_folder()` runs the full pipeline, deduplicating by SHA-256 hash of the file. Writes embeddings as raw little-endian f32 bytes into `doc_chunks_vec`. Re-indexing wipes old chunks (handles the sqlite-vec virtual table that has no FK).
  - `processes.py` — in-memory `process_id` → state registry for background-task polling.
- `backend/src/meetwit/routers/knowledge.py`
  - `POST /knowledge/index-folder` → returns `process_id`
  - `GET /knowledge/processes/{id}` → polling endpoint
  - `GET /knowledge/stats` → counts + last-indexed timestamp
  - `GET /knowledge/documents` → list with status
  - `DELETE /knowledge/documents/{id}` → wipe one
  - `DELETE /knowledge` → clear knowledge base
- `backend/src/meetwit/db.py` rewritten to use **`async_creator`** — the only reliable way to load sqlite-vec into every new aiosqlite connection (the SQLAlchemy `connect` event runs on a different thread than aiosqlite's worker and hits SQLite's threading rule).
- `backend/src/meetwit/main.py` lifespan now runs `run_migrations(settings.db_path)` on startup so cold-start "just works".
- Tests:
  - `test_indexing.py` — chunker/splitter unit tests + full end-to-end indexing of an MD + TXT folder using the real BGE model (downloads ~120 MB on first run; ~200s wall time)

## Manual API smoke test

```bash
cd backend
uv run python -m meetwit &
SIDECAR=$!

mkdir -p /tmp/meetwit-docs
echo "# Refund Policy" > /tmp/meetwit-docs/policy.md
echo "Customers can request refunds within 30 days." >> /tmp/meetwit-docs/policy.md

PID=$(curl -s -X POST http://127.0.0.1:5167/knowledge/index-folder \
  -H 'content-type: application/json' \
  -d '{"folder":"/tmp/meetwit-docs"}' | jq -r .process_id)
echo "process_id: $PID"

# Poll until finished
while true; do
  STATE=$(curl -s http://127.0.0.1:5167/knowledge/processes/$PID)
  echo $STATE | jq .
  echo $STATE | jq -e '.finished' >/dev/null && break
  sleep 1
done

curl -s http://127.0.0.1:5167/knowledge/stats | jq .

kill $SIDECAR
```
