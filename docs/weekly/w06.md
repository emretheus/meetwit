# Week 6 — Audio pipeline (ring buffer, mix, VAD)

**Status:** ✅ green

## Goal

Unify mic + system audio rings into a single 16 kHz mono stream with RMS-based ducking. Gate downstream consumers (ASR) on voice-activity detection.

## What landed

- `desktop/src-tauri/src/audio/mixer.rs`
  - `AudioMixer::start(mic_ring, system_ring)` spawns a `meetwit-mixer` thread
  - 50 ms windows (= 800 samples @ 16 kHz)
  - **RMS-based ducking**: system gain → 0.25 when mic RMS > 0.02
  - **Clip clamp**: `(mic + system * gain).clamp(-1.0, 1.0)`
  - **Energy VAD with hysteresis**: ON when RMS > 0.015, OFF when RMS < 0.008 for 6 frames (~300 ms hang)
  - Emits to two rings: `output_ring` (everything) and `voice_ring` (VAD-positive only)
  - `MixerStats` struct exposes `windows_processed`, `voice_windows`, `mic_rms`, `system_rms`, `last_mix_rms`, `is_voice`
- `AppState::mixer` slot, IPC commands: `mixer_start`, `mixer_stop`, `mixer_status`
- Removed `#[allow(dead_code)]` from `MicCapture::ring()` and `SystemCapture::ring()` — they're now real consumers' inputs

## VAD trade-off

The plan called for **Silero ONNX via `ort`** (ONNX Runtime). That brings in a ~150 MB native dep that needs `cmake` and lengthens the cold start. For V1 we ship an **energy VAD with hysteresis** — ~10 lines of code, no model file, no extra dep.

Tuning notes for the thresholds:
- `VAD_RMS_ON = 0.015` — quiet office speaker into a laptop mic
- `VAD_RMS_OFF = 0.008` — wider gap prevents flapping
- `VAD_HANG_FRAMES = 6` — 300 ms gives pauses room before declaring end-of-utterance

If false-positive/-negative rate is high on real meetings, swap in Silero in V1.1 — the `voice_ring` consumer doesn't care which VAD ran.
