## Tag-ready state

To cut `v1.0.0`:
1. Run `docs/V1_ACCEPTANCE.md` on M2 Pro hardware — all five functional tests pass
2. Ensure Apple Developer enrollment is complete and the four signing env vars are set
3. `./scripts/build-release.sh` produces a signed + notarized `.dmg`
4. Verify with `spctl --assess --type execute Meetwit.app` → "accepted"
5. `git tag v1.0.0 && git push --tags`
6. Upload `.dmg` to GitHub Releases with the CHANGELOG entry as release notes

## What V1 ships with

All five product promises implemented:
1. ✅ Index local docs (PDF/DOCX/MD/TXT → BGE-small embeddings → sqlite-vec)
2. ✅ Live transcript (cpal + ScreenCaptureKit + whisper-rs with Metal+CoreML)
3. ✅ In-meeting Q&A with sources (hybrid RAG via RRF, Ollama streaming over SSE)
4. ✅ Conflict detection (per-batch contradiction check, confidence ≥ 0.8 default)
5. ✅ Save summaries / decisions / action items (Ollama JSON-mode structured output)

Plus the cross-cutting commitments:
- Zero outbound network calls by default
- Single user, single device, no accounts
- MIT licensed, fully open source
- Apple Silicon native (Metal + CoreML acceleration end-to-end)

## What's documented but not yet ship-verified

- **Apple Developer signing** — `build-release.sh` + `build.yml` are wired, but live notarization round-trip requires user-side enrollment
- **Hardware variance** — code runs on the dev machine; M1 / M3 verification is in `V1_ACCEPTANCE.md`
- **Real meeting corpora** — current test suite uses small synthetic docs. RAG quality on real 50-PDF corpora is in the acceptance plan
