# Week 13 — Remaining screens + PyInstaller

**Status:** ✅ green

## Goal

Ship the missing screens that wire up to the working backend. Set up PyInstaller so the sidecar can be bundled into a .app.

## What landed

**Screens:**
- `desktop/src/components/SideNav.tsx` — left sidebar with 6 items (Home, Live meeting, Knowledge, Memory, Tasks, Settings) + active highlighting
- `desktop/src/routes/__root.tsx` — shell layout (sidebar + content area)
- `desktop/src/routes/knowledge.tsx` — `/knowledge`: stats grid, index-a-folder form with progress polling, indexed-documents list with delete, clear-all
- `desktop/src/routes/memory.tsx` — `/memory`: question input → streaming answer + always-open sources list
- `desktop/src/routes/tasks.tsx` — `/tasks`: filterable action items (all / open / done) with inline checkbox toggle
- `desktop/src/routes/settings.tsx` — `/settings`: backend health, Ollama detection + model list, Whisper model download instructions, privacy notes
- `desktop/src/routes/meeting.$id.summary.tsx` — `/meeting/$id/summary`: tabbed view (Overview / Decisions / Actions / Conflicts / Transcript) with "Process meeting" and "Detect conflicts" buttons that trigger the W11/W12 pipelines and poll for completion
- `desktop/src/routes/index.tsx` — Home rewritten: stats grid + recent meetings list

**Backend client (`desktop/src/lib/backend.ts`):**
- Extended with all remaining endpoint wrappers: `getMeeting`, `deleteMeeting`, `listDocuments`, `deleteDocument`, `clearKnowledge`, `getSummary`, `listDecisions`, `listActionItems`, `patchActionItem`, `listConflicts`, `triggerPostMeeting`, `triggerConflictDetection`, `llmStatus`, `indexProgress`

**Packaging:**
- `backend/meetwit-sidecar.spec` — PyInstaller `--onedir` spec
  - Bundles `sqlite_vec` package (including the `vec0.dylib` that PyInstaller's auto-discovery misses)
  - Bundles `migrations/` and `alembic.ini` so runtime migrations work in the .app
  - Targets `arm64`, console=True (stderr → Tauri log)
  - `codesign_identity=None` for downstream signing in `build-release.sh`
- `pyinstaller>=6.11` added to `[project.optional-dependencies].dev`
- `scripts/build-release.sh` rewritten to actually build:
  1. `uv run pyinstaller meetwit-sidecar.spec`
  2. Copy `backend/dist/meetwit-sidecar/` → `desktop/src-tauri/binaries/python-backend/`
  3. `pnpm tauri:build`
  4. Optional codesign + notarize via env vars `DEVELOPER_ID_APPLICATION`, `APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_PASSWORD`
