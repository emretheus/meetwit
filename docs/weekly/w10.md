# Week 10 — Live Assistant + Meeting UI

**Status:** ✅ green

## Goal

The in-meeting screen — live transcript on the left, ask-the-meeting Q&A on the right with citations. Tauri's transcript-update events flow from Rust → React → Python sidecar for persistence.

## What landed

**Backend:**
- `backend/src/meetwit/routers/meetings.py`
  - Full CRUD: `POST /meetings`, `GET /meetings`, `GET /meetings/{id}`, `PATCH /meetings/{id}`, `DELETE /meetings/{id}`
  - `POST /meetings/{id}/transcripts` (batch append from the frontend as Whisper emits segments)
  - `POST /live/ask` streams an SSE answer using the last `recent_seconds` of transcript + retrieved doc chunks
- `main.py` includes the new router

**Frontend:**
- `desktop/src/lib/backend.ts` — typed HTTP client for the Python sidecar with an SSE parser (`askMemory`, `liveAsk`, all knowledge + meeting endpoints)
- `desktop/src/lib/tauri.ts` extended: `onTranscriptUpdate`, `asrStart/Stop/Models`, `mixerStart/Stop/Status`
- `desktop/src/routes/meeting.live.tsx` — `/meeting/live` route:
  - Start button → creates a meeting → starts mic + system audio (best-effort) + mixer + ASR + transcript listener
  - Stop button → reverses all above + PATCH meeting to `completed`
  - Left pane: live transcript (each segment shows timestamp)
  - Right pane (420 px): Q&A input + streaming answer + sources list (default-open)
  - Each Whisper segment is also POSTed to `/meetings/{id}/transcripts` for persistence
- Home page `/` now has a prominent "● Start Live Meeting" link
