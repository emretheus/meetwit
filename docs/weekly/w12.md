# Week 12 — Conflict detection

**Status:** ✅ green

## Goal

Post-meeting pass that flags transcript spans contradicting company knowledge.
For each ~12-segment batch: retrieve top-5 doc chunks → ask LLM (JSON mode) for
contradictions → keep only `confidence ≥ 0.8`.

## What landed

- `backend/src/meetwit/services/conflicts.py`
  - `detect_conflicts(...)` walks transcripts in batches of 12, retrieves relevant docs per batch via `HybridRetriever`, calls Ollama with a `CONFLICT_SYSTEM` prompt that emphasizes conservatism, filters by confidence threshold, writes survivors to `conflicts` table
  - Idempotent: `DELETE FROM conflicts WHERE meeting_id = X` before processing
  - `ConflictProgress` dataclass tracks batches_done / batches_total / conflicts_found
- `backend/src/meetwit/routers/post_meeting.py`
  - `POST /conflicts/{meeting_id}/detect` — kicks off the detector, returns `{process_id}` (polled via the existing `/knowledge/processes/{id}` registry)
- Re-uses W11's `ConflictList` Pydantic schema and `structured_completion()` helper

## Prompt-engineering choices

The CONFLICT_SYSTEM prompt explicitly says:
- A conflict = an *explicit contradiction*, not vague tension
- Confidence ≥ 0.8 means "you'd bet on it"
- 0.5 = plausible-but-fuzzy
- 0.0-0.3 = omit
- Empty list if none

This is the main mitigation against the high false-positive rate flagged in Risk #10. If V1 acceptance still has too many FPs, raise the threshold to 0.9 — that's a one-line change in `services/conflicts.py`.

## Manual smoke test

```bash
# After post-meeting summary completes
PID=$(curl -s -X POST http://127.0.0.1:5167/conflicts/$MEETING_ID/detect \
  -H 'content-type: application/json' \
  -d '{"model":"qwen2.5:7b-instruct","confidence_threshold":0.8}' | jq -r .process_id)

while true; do
  STATE=$(curl -s http://127.0.0.1:5167/knowledge/processes/$PID)
  echo $STATE | jq '{stage,batches_done,batches_total,conflicts_found}'
  echo $STATE | jq -e .finished >/dev/null && break
  sleep 2
done

curl -s http://127.0.0.1:5167/conflicts/$MEETING_ID | jq .
```
