# Week 7 ‚Äî whisper-rs streaming transcription

**Status:** ‚úÖ green (compile + plumbing; live transcription requires a downloaded model)

## Goal

Real-time speech-to-text via whisper.cpp with Metal + CoreML acceleration. Transcripts stream to the frontend as `transcript-update` events. Frontend POSTs to the sidecar's `/meetings/{id}/transcripts` for persistence.

## What landed

- `desktop/src-tauri/src/asr/`
  - `model.rs` ‚Äî `ModelInfo` enum (TinyEn, BaseEn, SmallEn, MediumEn) with filename, label, and HuggingFace download URL
  - `engine.rs` ‚Äî `WhisperEngine::from_path(&Path)` loads ggml model; `transcribe(&[f32]) -> Vec<Segment>` returns timestamped text. Configured for English-only, greedy sampling, ~CPU-count threads (capped at 8)
  - `streamer.rs` ‚Äî `AsrStreamer` spawns `meetwit-asr` thread that pulls from a `SampleRing`, batches into 25 s windows with 3 s overlap, emits `TranscriptSegment { text, audio_start, audio_end, speaker }` via callback
- IPC commands: `asr_models`, `asr_start`, `asr_stop`, `asr_status`
- `AppState::asr` slot
- Frontend can subscribe via `listen("transcript-update", ‚Ä¶)` (wiring lands in Week 10 with the Meeting screen)
- **CI updated**: `brew install cmake` step before Rust build (whisper.cpp requires it)
- **bootstrap.sh updated** to flag missing `cmake`

## Build complexity added

`whisper-rs 0.13` with `metal` + `coreml` features pulls in whisper.cpp via `cmake`. Adds ~20 s to first build (then incremental). Required:
- `cmake` (added to bootstrap.sh + CI)
- `clang` from Xcode Command Line Tools (already required for Tauri)
- Metal framework (linked automatically by whisper-rs)

CoreML acceleration kicks in when a precompiled `.mlmodelc` sits next to the `.bin`. Not bundled in V1 ‚Äî Week 14 onboarding will offer the encoder model alongside the main `.bin`.
