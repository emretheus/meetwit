## Goal

Microphone capture working end-to-end on macOS. f32 mono audio at 16 kHz lands in a `SampleRing` shared across the app. Live RMS levels stream to the UI. WAV recording via `hound`.

## What landed

- `desktop/src-tauri/src/audio/`
  - `mod.rs` ‚Äî module surface
  - `ring.rs` ‚Äî `SampleRing` (parking_lot Mutex + VecDeque, capacity 30 s) + `rms()`
  - `wav.rs` ‚Äî `open_writer` (16 kHz mono PCM-int16) + `f32_to_i16` clamper + `write_samples`
  - `mic.rs` ‚Äî `MicCapture::start_default()` spawns a dedicated **meetwit-mic** thread that owns the non-`Send` `cpal::Stream`. The handle (`MicCapture`) is `Send + Sync` because it only holds `Arc<MicShared>` + `JoinHandle`.
- `AppState::mic()` slot in `state.rs`
- IPC commands: `mic_start`, `mic_stop`, `mic_status`, `mic_record_start`, `mic_record_stop`
- Frontend `MicControls` component on home page: start/stop, record/stop-record, live RMS bar, clip indicator
- Migrated workspace to **Rust edition 2024** (lets us use `if let ‚Ä¶ && let ‚Ä¶` chains)

## Audio details

- **Resampling** ‚Äî `rubato` `SincFixedIn` if device rate ‚âÝ 16 kHz (most MacBooks default to 48 kHz). 10 ms chunks at device rate; tail samples dropped per callback (acceptable for live capture).
- **Channels** ‚Äî averages all device channels into mono.
- **Sample formats** ‚Äî handles cpal's F32, I16, U16 native paths.
- **Recording** ‚Äî opens a `hound` `WavWriter` lazily on `mic_record_start`; samples flow into both the ring AND the writer until `mic_record_stop` finalizes.

## Edge cases & known limits

- **Device hotplug** ‚Äî not handled in V1 (cpal doesn't surface device-change events portably). Plug a new mic mid-meeting and you'll need to restart.
- **Permission denied** ‚Äî `start_default()` returns an error, which surfaces in the UI. Onboarding (Week 14) will deep-link to System Settings.
- **Resampler tail loss** ‚Äî Rubato wants fixed-size input chunks; ~10 ms of audio at chunk boundaries is dropped per callback. Negligible perceptually, fine for ASR.
- **Multi-instance** ‚Äî Tauri's `AppState` is a singleton; calling `mic_start` twice is a no-op.
