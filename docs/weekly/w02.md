# Week 2 — Sidecar lifecycle + IPC contract

**Status:** ✅ green

## Goal

Tauri Rust core auto-spawns the Python sidecar with port discovery, health-check polling, structured-log piping, graceful SIGTERM shutdown, and restart-on-crash with capped retries.

## What landed

- `desktop/src-tauri/src/sidecar/`
  - `mod.rs` — module surface
  - `manager.rs` — `SidecarManager::spawn` + `SidecarManager::supervise` watchdog + `SidecarHandle::shutdown`
  - `health.rs` — `wait_until_ready` with retry/backoff (50 ms × attempt, capped at 500 ms; 15 s overall deadline)
  - `client.rs` — typed HTTP wrapper `SidecarClient` (health, version)
  - `runtime.rs` — `runtime.json` writer/clearer at `~/Library/Application Support/Meetwit/runtime.json`
- `desktop/src-tauri/src/state.rs` — `AppState` with a `OnceLock<SidecarHandle>` slot
- `desktop/src-tauri/src/commands.rs` — added `backend_status` IPC command
- `desktop/src-tauri/src/lib.rs` — setup hook spawns sidecar async; `RunEvent::ExitRequested` cleanly shuts it down
- `desktop/src/lib/tauri.ts` — typed `backendStatus()`, `onBackendReady()`, `onBackendFailed()` event subscribers
- `desktop/src/routes/index.tsx` — Backend section showing live health badge + refresh
- Workspace clippy config relaxed for pedantic style lints (`doc_markdown`, cast lints) — kept `uninlined_format_args` because that actually catches readability bugs

## Lifecycle contract

```
Tauri app starts
    │
    ├─→ setup() hook
    │     ├─→ async task: SidecarManager::spawn(SpawnOptions::dev_default(workspace_root))
    │     │     ├─→ Command::new("uv").args(["run", "python", "-m", "meetwit"]).env("MEETWIT_PORT", "5167")
    │     │     ├─→ pipe stdout + stderr → log::info!()
    │     │     ├─→ wait_until_ready("http://127.0.0.1:5167", 15s)
    │     │     ├─→ write runtime.json {pid, host, port, started_at}
    │     │     └─→ emit "backend-ready" event
    │     └─→ async task: SidecarManager::supervise() — restart loop, max 3 consecutive failures
    │
    ├─→ Frontend renders, listens for "backend-ready"
    │
    └─→ App quit (RunEvent::ExitRequested)
          └─→ SidecarHandle::shutdown():
                ├─→ SIGTERM
                ├─→ wait 3 s
                ├─→ SIGKILL if still alive
                └─→ clear runtime.json
```

## Known limitations / deferred

- **Restart loop is best-effort.** The `Arc::try_unwrap` in `supervise()` only swaps state when no other clones exist; for V1 dev this is fine, but a stricter pattern arrives once we add tests.
- **Port collision detection** — we use the fixed 5167. Week 13 adds a "find free port if taken" fallback.
- **Sidecar binary path in release builds** — currently dev-only (`uv run python -m meetwit`). Week 13 wires the PyInstaller binary path.
- **Integration tests** — no automated test that spawns + kills the real sidecar yet. Add when the schema lands (Week 3) so we can also verify migrations run on cold start.
